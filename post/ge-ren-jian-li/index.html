项目经验

商户中心（2016.4 - 2017.4）

  • 简介：支撑餐饮门店点餐系统的商户后台设置系统
  • 技术栈：Stapes.js, jQuery, Handlebars, Grunt

前端单页面应用，MVC 思想在前端的早期探索和实践。数据模型层（M）主要负责业务数据的处理，示图层（V）主要负责数据的呈现，使用 Handlebars 进行模板渲染，jQuery 控制页面响应逻辑，控制层（C）主要负责调控，使用 Stapes 实现类的封装与继承，不同层之间通过事件机制通讯。项目使用 Grunt 构建打包。
该项目共有 5 个前端共同维护，各自负责不同的业务模块。

新商户中心（2016.12 - 2018.4）

  • 简介：商户集成管理系统，包含商户后台设置系统、供应链系统、加盟系统等多个子系统。
  • 技术栈：React, Redux, webpack, koa

由于公司业务的迅速扩张，原有前端架构难以支撑，故采用当时流行的 React 技术栈对项目重构。
新商户中心采用微前端架构，不同的业务子系统独立开发、打包、发布，同时引入了 nodejs server，负责前端接口的 grpc 代理调用以及服务端渲染等。
该项目约有三十个前端负责维护，按小组分模块维护不同的子系统，我主要负责基础框架、组件库以及部分业务子模块的维护。

系统监视器（2017.6 - 2018.4）

  • 简介：支撑微前端架构下新商户中心项目的运维系统。
  • 技术栈：React, MobX, webpack, koa

为配合微前端架构下新商户中心各个子系统的独立发布与管理，开发了该运维系统，可对线上系统的运行状态进行实时监控，同时实现版本的发布与回退等功能。
该项目较为小巧且维护人数较少，故尝试引入 MobX 进行数据管理，相比 Redux 使用更灵活。

云财务系统（2018.4 至今）

  • 简介：对接餐饮和供应链的财务一体化系统。
  • 技术栈：Vue.js, webpack

该项目为纯前端项目，项目框架直接使用 Vue Cli 搭建，组件库是基于 Element 二次开发的。
共有 5 个前端负责该项目的开发和维护，我主要负责基础框架和组件库的维护工作。

打印服务（2018.11 至今）

  • 简介：实现财务数据到打印表单的转化。
  • 技术栈：Express, pdfmake

该项目是一个 nodejs 的小型服务端项目，使用 typescript 编写，提供打印相关的接口，从客户端接收打印数据并选择合适的模板进行渲染，输出 PDF 文件返回给客户端以供打印。
这种打印方式没有直接控制打印机，是一种基于浏览器的纯前端技术打印解决方案。

帮助中心（2019.9 - 2020.6）

  • 简介：为财务系统用户提供操作指引。
  • 技术栈：Vue.js

该项目包含了文档上传和文档呈现两个子项目，文档上传实现了 DOCX、PPTX 等格式的文档到 HTML 的转换，文档呈现则实现了静态文档阅读、幻灯片播放以及指引教程播放等功能。

财务审批小程序（2020.7 至今）

  • 简介：对接财务软件的移动审批小程序。
  • 技术栈：uni-app

该项目使用 uni app 实现多平台小程序的统一开发维护，配合各平台开发 cli 工具实现统一发布。
